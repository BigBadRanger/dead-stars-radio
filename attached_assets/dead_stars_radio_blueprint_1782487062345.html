What it is
The concept
Dead Stars Radio is a web app where users tune into real stars using actual NASA light curve data. Each star is sonified into an audio signal — its brightness over time becomes pitch, rhythm, and texture. A generated image shows the star in multiple spectra. An AI narrator explains what you're hearing, including whether the star still exists. Some of what you're listening to is a fossil — light and sound from something already gone.
Data strategy — recommended approach
Hybrid: Curated dataset + live NASA API
Pre-load 25 hand-picked stars from NASA's MAST archive (light curve CSVs — free, no auth needed for most). These power your demo reliably. Also wire the NASA Exoplanet Archive API for live search — when someone types a star name, it fetches real data. Curated = demo-safe. Live API = the jaw-drop moment.
System architecture
Layer 1 — Data

Star Data Ingestion

NASA MAST API for light curves (brightness over time). NASA Exoplanet Archive for stellar parameters (mass, age, distance, spectral class, alive/dead status). Pre-process into a normalized format: time series of flux values + metadata object per star.

Layer 2 — Sonification

Light Curve → Audio Engine

Map flux (brightness) to pitch and amplitude. Map periodicity (pulsing stars) to rhythm. Map spectral class (O, B, A, F, G, K, M) to instrument timbre — hot blue stars = high-frequency tones, cool red dwarfs = low resonant hum. Use Tone.js in the browser for real-time audio synthesis from the data. No pre-rendered audio files — it generates live from the numbers.

Layer 3 — Vision

Multimodal Star Imagery

Use real NASA image assets where available (most famous stars have public domain imagery). For lesser-known stars, generate an image using the stellar parameters as prompt inputs — spectral class, temperature, and size drive the visual aesthetic. Show 2–3 views: visible light, X-ray, infrared.

Layer 4 — AI Narration

Claude Agent Narrator

A Claude agent receives the star's metadata + sonification parameters and generates a real-time narration: what you're hearing, why it sounds this way, the star's life story, and crucially — whether it still exists. If the star is dead, the agent narrates the transmission lag. "The light you see left this star 640 years ago. It may no longer exist."

Layer 5 — UI

The Radio Interface

Aesthetic: a retro-futurist deep space radio. Star selector (search or browse the curated dial). Waveform visualizer that pulses to the audio. Star portrait with spectral overlays. Narration panel that streams in as audio plays. "Transmission age" indicator showing how old the signal is. Dead star indicator — a subtle red glow if the star no longer exists.

Tech stack
React (frontend) Tone.js (audio synthesis) NASA MAST API NASA Exoplanet Archive Claude API (narration agent) Stable Diffusion / DALL-E (star imagery) D3.js (waveform viz) Python backend (data pre-processing) Vercel or Netlify (deploy)

Live demo arc (5–10 min)

Min 0–1: Open in silence. Pull up the radio UI. No explanation. Tune to Betelgeuse. Audio starts — a low, slow, pulsing hum. Let it play for 15 seconds. Then say: "What you're hearing left this star 700 years ago. Betelgeuse is expected to go supernova. It may already be dead. We won't know for 700 years."

Min 2–4: Tune to a contrasting star — a hot blue O-class star. The sound completely changes. Explain the sonification mapping live. Show the waveform responding. Read a line from the AI narration as it streams in.

Min 5–7: Explain the architecture — NASA data, Tone.js, Claude narration agent. Keep it tight, one slide max.

Min 8–10: Live audience moment — ask someone to shout out any star they know. Search it live. Let the signal play. Let the AI narrate it. If it's a dead star, that moment lands on its own.
